# Getting Started

## Installation

To use `ChatOps4s` with Slack, install the following dependency

<SbtDependency moduleName={"chatops4s-slack"} />

## Picking an `sttp` Backend

You must install an `sttp` backend which `ChatOps4s` will use to make HTTP requests on your behalf. View your choices of backends on `sttp`'s [documentation](https://sttp.softwaremill.com/en/latest/backends/summary.html).

## Picking a `tapir` Server Interpreter

You must install a `tapir` server interpreter to allow `ChatOps4s` to receive HTTP requests from Slack. View your choices of server interpreters on `tapir`'s [documentation](https://tapir.softwaremill.com/en/latest/server/overview.html).

## Setting up your Server

Once you have picked a server interpreter and `sttp` backend, import the `SlackGateway` class to set up your server. In the following example, we create a `ChatOps4s` server using the `http4s` interpreter (using `ember`) and `cats` backend for `sttp`:

```scala file=./main/scala/slack/docs/GettingStarted.scala start=start_doc end=end_doc
```

## Configuration

The `SlackConfig` requires:

- `botToken`: Your Slack bot token, which you can obtain from your [Slack app configuration](https://api.slack.com/apps)
- `signingSecret`: Your Slack signing secret, used to verify requests from Slack
- `port`: The port your server will listen on (optional, defaults to 3000)

## Environment Variables

It's recommended to set your Slack credentials as environment variables:

- `SLACK_BOT_TOKEN`: Your bot token (starts with `xoxb-`)
- `SLACK_SIGNING_SECRET`: Your app's signing secret
- `PORT`: Server port (optional)

## Next Steps

Once your server is running, you can:
- Send messages to channels
- Create interactive buttons
- Handle user interactions
- Send threaded messages
